## Summary

Build a responsive web application for splitting bills and tracking payments among group members. The app will allow organizers to create bills with participants, generate shareable links, and track payment status via a dashboard. A manual/simulated payment flow will be used to keep the process frictionless.

## Technical Context

**Language/Version**: TypeScript / Node.js 20+

**Primary Dependencies**: Next.js 14+ (App Router), Supabase (Auth/DB), Tailwind CSS, Lucide React, Shadcn/UI

**Storage**: Supabase (PostgreSQL)

**Testing**: Vitest (Unit/Integration), Playwright (E2E)

**Target Platform**: Web (Responsive, optimized for mobile browsers)

**Project Type**: Web application

**Performance Goals**: < 2s initial load, < 500ms dashboard updates, responsive at 320px+

**Constraints**: Mobile-first, WhatsApp web view compatibility, no-auth participant flow

**Scale/Scope**: MVP for small to medium group collections (up to 50 participants per bill)
